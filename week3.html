Sprint Retrospective

Week 3 Sprint Retrospective

What went well?

I began ideating potential solutions and narrowed down my focus to food storage and expiry tracking. I created a one-sentence pitch and started mapping out user flows and future journey scenarios also created a site map. This helped me visualise the core functionality of the platform. I felt more confident in the project direction and successfully met my Sprint goal of moving from insights to early concepts. I have also started to create my lo-fi prototypes which was within my plans.

What didn't go well?

I felt a bit stuck choosing between multiple solution ideas. I spent some time on if I wanted to add features like recipe generation, which slowed down decision-making. Some design elements also started to feel too broad again.

What can be improved?

I learned the value of focusing on a single, strong idea instead of trying to combine too many features which may make the project too complex and not focusing on the main pain point. Simplicity and clarity are more impactful at this stage. I also need to trust my user insights more when prioritising features.

What actions will I take for the next sprint?

  • Start to create brand identity
  • Start thinking about the Proof of Concept interaction
  • Keep the solution tightly aligned with user pain points and scope
  • Conduct user testing for lo-fi and then start to create hi-fi prototype
